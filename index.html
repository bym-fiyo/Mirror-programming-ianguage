[ Философия ]

// Почему Mirror существует

"Я создал Mirror не потому, что мир нуждался в ещё одном языке программирования. Я создал его, потому что мир нуждался в языке, который не заставляет страдать."
"🪞 Зеркало отражает ваш интеллект. Так что если код будет тупым, не обижайтесь!" — Древняя мудрость Mirror

Mirror Language родился из фрустрации. Фрустрации от языков, где нужно написать 10 строк кода, чтобы сделать простую вещь. От языков, где синтаксис выглядит как случайный набор скобок. От языков, где ты тратишь больше времени на борьбу с компилятором, чем на решение задачи.

// Мои принципы

📖

Читаемость прежде всего

Код на Mirror должен читаться как обычный текст. Если ваш бабушка не может понять, что делает код — я проиграл.

⚡

Минимум кода, максимум результата

Зачем писать 5 строк, если можно написать 1? Я верю в лаконичность без потери понятности.

🤝

Дружелюбность к новичкам

Первый язык программирования не должен быть испытанием на прочность. Mirror создан для того, чтобы учиться было приятно.

🔓

Открытость

Mirror — open source проект. Я верю, что лучшие языки создаются сообществом, а не в закрытых лабораториях.

// Манифест Mirror

// Я верю, что...

1. Код должен быть читаемым, как текст
2. Ошибки должны помогать, а не пугать
3. Простота не означает слабость
4. Хороший язык учит хорошему коду
5. Программирование доступно каждому

// Я не верю в...

1. Магические скобки
2. Непонятные ошибки компилятора
3. Сложность ради сложности
4. Элитарность в программировании
5. Языки, которые требуют PhD для понимания

[ Ценности ]

// Что для меня важно

01

🔍 Прозрачность

Никакой магии. Если что-то происходит, вы должны понимать почему. Mirror не скрывает от вас то, что происходит под капотом. Каждая функция, каждая операция — открыта и понятна.

var x = 10 // Просто и понятно

Почему это важно: Когда вы видите, как работает код, вы можете его улучшить, исправить и доверять ему.

02

📐 Последовательность

Если вы выучили одну часть языка, остальные части будут логичным продолжением. Никаких исключений из правил. Все функции работают одинаково, все типы данных ведут себя предсказуемо.

str_upper(s) // Все функции работают одинаково

Почему это важно: Последовательность снижает когнитивную нагрузку. Вы тратите меньше времени на запоминание и больше на решение задач.

03

⚡ Практичность

Я добавляю только то, что действительно нужно. Никаких академических фич "на всякий случай". 95+ функций в стандартной библиотеке покрывают 99% задач разработчика: работа со строками, файлами, криптографией, JSON и ОС.

use file, crypto, json, math // Только то, что нужно

Почему это важно: Меньше значит больше. Простой язык с мощными возможностями — это баланс, к которому я стремлюсь.

04

🤝 Сообщество

Mirror создан разработчиками для разработчиков. Каждый вклад важен, каждая идея имеет значение. Я верю, что лучшие языки создаются сообществом, а не в закрытых лабораториях.

// Присоединяйтесь!

Почему это важно: Один в поле не воин. Вместе я могу создать язык, который поможет миллионам разработчиков.

05

🎓 Доступность

Программирование должно быть доступно каждому, независимо от возраста, образования или опыта. Mirror создан так, чтобы первый язык программирования не был испытанием на прочность.

echo("Hello, World!") // Первая строка кода

Почему это важно: Когда язык понятен новичкам, он становится мощным инструментом обучения. Хороший код начинается с хорошего языка.

06

🔓 Открытость

Исходный код интерпретатора открыт и доступен. Вы можете изучить его, улучшить, предложить изменения. Я верю в силу open source и коллективного разума.

gcc -o mirror mirror.c // Компиляция из исходников

Почему это важно: Открытый код — это доверие. Вы знаете, что делает интерпретатор, и можете это проверить.

// Управляй потоком как бог

// If / Else

var age = 20

if age >= 18 {
    echo("Adult")
} else {
    echo("Minor")
}

# Тернарный оператор
var status = age >= 18 ? "adult" : "minor"

// Switch / Case

var day = 3

switch day {
    case 1: echo("Mon")
    case 2: echo("Tue")
    case 3: echo("Wed")
    default: echo("Other")
}

// For Loop

# Диапазон чисел
for i in range(5) {
    echo(i)  # 0,1,2,3,4
}

# Массив
for item in [1,2,3] {
    echo(item)
}

// While Loop

var count = 5

while count > 0 {
    echo(count)
    count--
}

# Бесконечный цикл
loop {
    if done { stop }
}

// Try / Catch

try {
    var result = 10 / 0
} catch (e) {
    echo("Error: " + str(e))
} finally {
    echo("Cleanup")
}

# finally выполняется всегда
# Как дедлайн в пятницу вечером

// Control

# Прерывание
for i in range(10) {
    if i == 5 { stop }
}

# Продолжение
for i in range(10) {
    if i % 2 == 0 { next }
    echo(i)
}

[ Зеркальные функции ]

// Уникальная особенность Mirror — выполнение в обратном порядке

🪞

Mirror — первый язык программирования с поддержкой зеркальных функций. Они выполняют код в обратном хронологическом порядке — от последнего оператора к первому.

// Обычная функция

func normal():
    echo("first")
    echo("second")
    echo("third")

normal()
# Вывод:
# first
# second
# third

// Зеркальная функция

mirrorfunc mirror():
    echo("first")
    echo("second")
    echo("third")

mirror()
# Вывод:
# third
# second
# first

// Когда использовать mirrorfunc?

🔄 Обратные операции

Автоматическое выполнение действий в обратном порядке — идеально для отмены операций, rollback транзакций, очистки ресурсов.

🎯 Декларативный код

Описывайте логику в естественном порядке, а выполнение произойдёт зеркально — полезно для стековых операций.

🧪 Тестирование

Тестируйте инварианты: создайте состояние и проверьте обратное разрушение — отличный способ отладки.

🎨 Креативное программирование

Создавайте необычные алгоритмы и визуальные эффекты с помощью обратного выполнения.

// Практический пример: управление ресурсами

# Зеркальная функция для очистки ресурсов
mirrorfunc cleanup():
    echo("Closing file...")
    echo("Releasing memory...")
    echo("Disconnecting from DB...")
    echo("Cleanup complete!")

cleanup()
# Вывод:
# Cleanup complete!
# Disconnecting from DB...
# Releasing memory...
# Closing file...

// Синтаксис

mirrorfunc function_name(param1, param2):
    # Тело функции
    # Выполняется снизу вверх
    ret value  # Возврат тоже работает
💡 Важно: Зеркальные функции используют ключевое слово mirrorfunc (слитно). Отступы работают так же, как в обычных функциях — 4 пробела на уровень вложенности.
